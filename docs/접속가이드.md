# 그룹 챌린지 앱 접속 가이드

## 🚀 빠른 시작

### 1. 필수 요구사항 확인

다음이 설치되어 있어야 합니다:
- **Node.js**: 20.x 이상
- **pnpm**: 9.x 이상
- **PostgreSQL 데이터베이스** (Supabase 권장)

### 2. 의존성 설치

```bash
pnpm install
```

### 3. 환경 변수 설정

`.env.local` 파일을 생성하고 다음 변수들을 설정하세요:

```bash
# Database (필수)
DATABASE_URL="postgresql://사용자명:비밀번호@호스트:포트/데이터베이스명"
DIRECT_URL="postgresql://사용자명:비밀번호@호스트:포트/데이터베이스명"

# NextAuth (필수)
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="openssl rand -base64 32 명령어로 생성한 랜덤 문자열"

# OAuth (선택사항 - 소셜 로그인 사용 시)
KAKAO_CLIENT_ID="카카오 개발자 콘솔에서 발급"
KAKAO_CLIENT_SECRET="카카오 개발자 콘솔에서 발급"
GOOGLE_CLIENT_ID="구글 클라우드 콘솔에서 발급"
GOOGLE_CLIENT_SECRET="구글 클라우드 콘솔에서 발급"
NAVER_CLIENT_ID="네이버 개발자 센터에서 발급"
NAVER_CLIENT_SECRET="네이버 개발자 센터에서 발급"

# Vercel Blob Storage (이미지 업로드용 - 선택사항)
BLOB_READ_WRITE_TOKEN="Vercel Blob Storage에서 발급"

# Supabase (실시간 기능용 - 선택사항)
NEXT_PUBLIC_SUPABASE_URL="Supabase 프로젝트 URL"
NEXT_PUBLIC_SUPABASE_ANON_KEY="Supabase Anon Key"
SUPABASE_SERVICE_ROLE_KEY="Supabase Service Role Key"
```

#### NEXTAUTH_SECRET 생성 방법

```bash
# Windows (PowerShell)
[Convert]::ToBase64String([System.Text.Encoding]::UTF8.GetBytes([System.Guid]::NewGuid().ToString() + [System.Guid]::NewGuid().ToString()))

# Mac/Linux
openssl rand -base64 32
```

### 4. 데이터베이스 설정

#### Supabase 사용 시 (권장)

1. [Supabase](https://supabase.com)에서 프로젝트 생성
2. 프로젝트 설정 > Database > Connection string에서 연결 문자열 복사
3. `.env.local`에 `DATABASE_URL`과 `DIRECT_URL` 설정

#### 로컬 PostgreSQL 사용 시

1. PostgreSQL 설치 및 실행
2. 데이터베이스 생성:
```sql
CREATE DATABASE group_challenge;
```
3. `.env.local`에 연결 정보 설정

### 5. Prisma 설정

```bash
# Prisma 클라이언트 생성
pnpm db:generate

# 데이터베이스 마이그레이션 (테이블 생성)
pnpm db:migrate
```

### 6. 개발 서버 실행

```bash
pnpm dev
```

### 7. 브라우저에서 접속

개발 서버가 실행되면 다음 URL로 접속:

**http://localhost:3000**

## 🔧 문제 해결

### 포트 3000이 이미 사용 중인 경우

다른 포트로 실행:
```bash
pnpm dev -- -p 3001
```

### 데이터베이스 연결 오류

1. `.env.local`의 `DATABASE_URL` 확인
2. 데이터베이스 서버가 실행 중인지 확인
3. 방화벽 설정 확인

### Prisma 마이그레이션 오류

```bash
# 마이그레이션 초기화 (주의: 기존 데이터 삭제됨)
pnpm db:push --force-reset

# 또는 새로 마이그레이션
pnpm db:migrate
```

### 의존성 설치 오류

```bash
# 캐시 삭제 후 재설치
rm -rf node_modules pnpm-lock.yaml
pnpm install
```
